# 问题一：bert 文本分类和 实体识别有什么关系，分别使用什么loss？

答：

bert 文本分类使用的loss为：CrossEntropyLoss，将模型预测的类别分布与真实类别（one-hot）计算交叉熵，只针对`[CLS]`位置的输出进行损失计算。

bert 实体识别使用的loss为：交叉熵损失（CrossEntropyLoss），但按token计算，对每个非特殊token（通常忽略填充token和特殊token）的预测与真实标签计算交叉熵，然后将所有有效位置的损失取平均或求和。在实现中，常将需要忽略的位置的标签设为`-100`，在损失函数中自动忽略。

# 问题而：多任务训练 loss = seq_loss + token_loss 有什么坏处，如果存在训练不平衡的情况，如何处理？

答：

**坏处：**

* 损失量级不一致
  
  * 不同任务的损失函数值可能相差很大（例如，交叉熵损失的大小取决于类别数和任务难度）。如果直接相加，量级较大的任务会主导梯度更新，导致量级较小的任务几乎不被优化。
  
  * 例如，句子级分类的损失通常在 `[0, logN]` 之间，而逐 token 分类的损失可能累加后远大于前者（因为涉及多个 token），从而造成任务失衡。

* 梯度冲突与优化方向不一致
  
  * 两个任务的梯度方向可能不同，甚至相反。直接相加可能导致参数在相互矛盾的方向上更新，使模型难以收敛，或在两个任务间折衷，最终效果都不理想。

* 收敛速度不匹配
  
  * 不同任务的学习难度不同，一个任务可能很快收敛，而另一个任务仍需大量训练。简单的损失和无法动态调整任务重要性，可能导致早熟或欠拟合。

* 过拟合风险
  
  * 多任务模型通常共享底层参数，如果任务相关性低或数据不足，共享参数可能无法充分捕捉各自特征，反而增加过拟合风险（尤其当某个任务数据量较少时）。

* 超参数敏感
  
  * 直接相加等同于给两个任务分配了固定的相对权重（均为1）。这个权重往往不是最优的，需要手动调整，而调参过程可能耗时且难以找到最佳平衡。

**处理不平衡的方法：**

* 加权损失
  
  * 手动调整：根据验证集性能调整权重（如 `L_total = α·L_seq + β·L_token`），通过网格搜索或随机搜索找到合适比例。
  
  * 不确定性加权：将任务权重建模为可学习的参数，基于任务噪声（不确定性）自动调整权重，公式为  
    `L_total = Σ (1/(2σ_i²))·L_i + log σ_i`，其中 σ_i 为任务 i 的观测噪声。

* 梯度平衡技术
  
  * 梯度归一化：在更新前对各任务的梯度进行归一化（如除以梯度范数），防止主导任务梯度掩盖其他任务。
  
  * PCGrad：将冲突梯度投影到彼此的法向方向，减少梯度干扰。
  
  * MGDA：寻找 Pareto 最优方向，平衡多目标优化。

* 任务特定学习率与模块化
  
  * 为不同任务的输出头设置独立的学习率，或对共享层和任务特定层使用不同学习率。
  
  * 设计模型结构时，保持底层共享，上层分离，减少任务干扰。

* 动态训练策略
  
  * 交替训练：在每个 batch 或 epoch 中交替更新不同任务，避免梯度累积冲突。
  
  * 课程学习：先训练简单任务，再引入复杂任务，逐步增加难度。

* 数据层面调整
  
  * 如果某任务数据量过少，可通过数据增强、重采样（过采样或欠采样）平衡数据量，间接平衡损失贡献。
  
  * 使用带权重的损失函数（如 Focal Loss）调整单个任务的难易样本权重，提升任务内部稳定性。

* 正则化与早停
  
  * 对任务特定参数施加更强的正则化（如 Dropout、权重衰减），防止过拟合。
  
  * 使用早停机制监控验证集上所有任务的综合表现，避免某个任务过拟合。
