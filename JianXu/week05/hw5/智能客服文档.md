# 文本语义匹配模型设计与实验总结

## 1. 任务背景与目标

本项目的目标是 **中文用户问题的语义匹配 / 意图识别**。给定一条用户输入问题，模型需要判断它最接近哪一个「标准问题」。

整体思路不是直接做多分类，而是：

* 先将所有问题编码为向量（Sentence Embedding）
* 在向量空间中，通过 **相似度搜索** 找到最相近的标准问题

文本匹配任务中的 **表示学习（Representation Learning） + 相似度检索** 任务。

---

## 2. 数据设计

### 2.1 数据文件结构

数据主要由三部分组成：

1. **schema.json**：标准问题（意图）到编号的映射
2. **train.json**：训练数据（用于构建知识库 + 采样训练）
3. **valid.json**：验证 / 测试数据（用于评估）

### 2.2 schema 设计

```json
{
  "话费查询": 0,
  "积分查询": 1,
  "呼叫转移设置": 2,
  ...
}
```

* 每一个 key 表示一个 **标准意图**
* value 是模型内部使用的 **label index**

该设计保证：

* 语义类别稳定
* 模型与具体文本解耦

---

### 2.3 训练数据设计（train.json）

训练集以 **“标准问 + 多个等价问”** 的形式组织：

```json
{
  "questions": ["查一下我的话费", "我想查话费", "话费还有多少"],
  "target": "话费查询"
}
```

含义：

* `questions`：同一语义下的不同自然语言表达
* `target`：对应的标准意图

在加载数据时：

* 每个 question 会被编码成 token id 序列
* 按照 `target` 被归入同一个 **知识库 knwdb[label]** 中

> 训练阶段并不直接使用 label 做分类，而是构建「正负样本对」。

---

### 2.4 测试数据设计（valid.json）

测试集为逐条样本：

```json
["我这个月用了多少话费", "话费查询"]
```

加载后得到：

* `input_ids`：句子 token id
* `label_idx`：标准问题编号

测试阶段用于：

* 将问题编码为向量
* 与知识库中所有问题向量计算相似度
* 判断 Top-1 是否命中正确意图

---

## 3. 文本表示与编码方式

### 3.1 分词与编码

当前实现支持 **字符级建模**：

* 每个汉字作为一个 token
* 使用 `chars.txt` 构建 vocab
* 未登录字符映射为 `[UNK]`

```text
句子 → 字符序列 → token ids → padding → (max_length)
```

Padding 策略：

* 截断超过 `max_length` 的部分
* 不足部分补 0

Embedding 层设置：

```python
nn.Embedding(vocab_size + 1, hidden_size, padding_idx=0)
```

---

## 4. 训练样本构造方式（Triplet）

### 4.1 三元组采样策略

训练阶段使用 **Triplet Loss**，每个样本包含：

* Anchor（a）：某一标准问下的问题
* Positive（p）：同一标准问的另一个问题
* Negative（n）：不同标准问的问题

```text
a, p ∈ 同一意图
n ∈ 不同意图
```

在 `random_train_sample()` 中动态生成：

* 每个 epoch 不固定训练样本
* 提升样本多样性
* 减少过拟合
