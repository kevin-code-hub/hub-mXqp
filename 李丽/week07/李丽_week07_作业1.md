### 作业1：阅读 02-joint-bert-training-only 代码并回答问题
1. BERT 文本分类和实体识别有什么关系，分别使用什么 loss？
- 文本分类和实体识别，这两个任务在代码中是**联合训练（Joint Training）**的。它们共用同一个 BERT Encoder 来提取句子的特征
  - 文本分类（意图识别） ：使用的是 BERT 输出的 [CLS] 向量 （整句的语义表示），接一个全连接层进行分类。
  - 实体识别（槽位填充） ：使用的是 BERT 输出的 每个 Token 的向量 （字级别的语义表示），接全连接层进行序列标注（BIO标签预测）
- 分别使用了什么Loss：这两个用的Loss是一样的，叫 CrossEntropyLoss（交叉熵损失）
  - 具体计算 ：
    - 意图 Loss ：算的是整句话预测意图与真实标签的误差。
    - 实体 Loss ：算的是每个字（Token）预测标签与真实标签的误差，并且会忽略掉 padding 部分（即只计算有效字符的 Loss）。


2. 多任务训练 loss = seq_loss + token_loss 有什么坏处？如果存在训练不平衡的情况，如何处理？
- 直接相加的坏处：
  - 量级不一致（梯度主导） ：
    - 代码中直接把两个 Loss 相加。但通常 实体识别 Loss ( token_loss ) 会比 意图识别 Loss ( seq_loss ) 大很多（因为实体识别是对几十个字求 Loss，而意图只对一句话求 Loss）。
    - 这就导致模型训练时，梯度主要被实体识别任务主导，模型会拼命优化实体识别，而忽略了意图识别的优化。
  - 收敛速度不同步 ：一个任务可能早就学好了（过拟合），另一个还在艰难学习（欠拟合），直接相加无法兼顾两者的进度

- 处理办法：加权求和
  - 给两个 Loss 加上权重系数
  - 如果实体识别 Loss 太大，就给它乘个小数（比如 0.1），或者给意图 Loss 乘个大数，强行把两个 Loss 拉到同一个数量级。
